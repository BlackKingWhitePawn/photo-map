# Photo Map v0.8.2

Date: 2026-07-17

## Added

* No new user-facing features.

## Changed

* Singleton stored cluster rows are normalized to single photo map items before rendering.
* Map marker rendering now uses the Compose overlay only; duplicate MapLibre marker sources are cleared on refresh.

## Fixed

* A photo stored as a cluster with `photoCount=1` no longer appears as both a cluster and a single photo marker.
* Removed the extra marker outline/artifact caused by drawing the same photo through both MapLibre layers and the Compose overlay.
