# Photo Map v0.9.0

Date: 2026-07-18

## Added

* Semantic trip segmentation from locally indexed photos with valid date and geotag data.
* Local trip tables in `photo_index.db` for trips, trip-photo links, and trip destinations.
* Stored trip place names in `photo_index.db`, resolved during trip segmentation.
* Navigation Compose app routing with destinations for the main map, settings, trip map, and trip detail screen.
* `Поездки` FAB on the main map.
* Separate dark MapLibre trip map.
* `TRIP_MAP_STYLE_URL` build config, defaulting to `https://tiles.openfreemap.org/styles/dark`.
* Trip map markers rendered as photo thumbnails with `+N` photo-count badges.
* Trip detail screen opened by tapping a trip marker.
* Platform geocoder place names in the trip detail title, with trip dates in the subtitle.
* Smoothed chronological trip route line on the trip detail map, with photo points and a start-to-end gradient.
* Clickable photo thumbnails on route points that scroll the lower trip gallery to the selected photo.
* Read-only opening of trip grid photos in the default Android gallery.
* Grid gallery of the selected trip photos below the route map.
* Long right-edge trip timeline scrubber that snaps to trip points and shows semi-transparent trip labels beside the marks.
* Collapsed `...` entry button for the trip timeline scrubber.
* Wheel-like trip timeline dragging with minimal edge padding.
* Active trip timeline point highlighted with accent color and a border; single-trip timelines render as a point without a line.
* Matching right-edge photo scrubber on the trip detail screen for jumping between photos inside a trip.
* Trip detail centering action that resets route map zoom to the whole trip.

## Changed

* Android version metadata is now `versionName=0.9.0`, `versionCode=15`.

## Fixed

* Main map viewport refresh now runs after scan, camera fit, and returning from trip screens so normal photo markers appear without requiring a manual map move.
* Selected trip map zoom and center are preserved when returning from a trip detail screen.
* Accidental overlay taps are blocked for a short time after map gestures so pinch/drag/zoom does not open trips or clusters.
