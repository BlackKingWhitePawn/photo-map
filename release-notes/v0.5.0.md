# Photo Map v0.5.0

Дата: 2026-07-15

Релиз добавляет новый режим карты: heatmap-группы вместо чистых точек, миниатюрные маркеры для плотных групп фотографий и настройки порога их появления. Также добавлено управление долгим индексированием: пауза, продолжение и отмена.

## Добавлено

* Heatmap-отображение фотографий на MapLibre-карте.
* Маркеры с миниатюрой представительной фотографии для групп, достигших порога.
* Подпись миниатюрных маркеров в формате `+N фото`.
* Экран настроек с изменением порога появления миниатюр.
* Сохранение порога heatmap-миниатюр в локальных настройках приложения.
* Пауза, продолжение и отмена активного индексирования.
* Кнопка отзыва доступа через системные настройки приложения.
* Иконка настроек для кнопок перехода в настройки.
* Локальный стек навигации с возвратом на предыдущий экран по системной кнопке `назад`.

## Изменено

* Карта остается доступной во время индексирования и больше не показывает пустое состояние `нет фотографий`, если сканирование уже идет.
* Уже обработанные batch-результаты индексирования сразу попадают в `UiState`, поэтому карта и список наполняются до завершения полного прохода.
* Основная кнопка доступа меняет состояние:
  * `Предоставить доступ`, если доступа нет;
  * `Отозвать доступ`, если доступ уже выдан.
* Отмена сканирования не удаляет уже сохраненный локальный индекс.
* Локальный Gradle cache `.gradle-home/` исключен из Git.

## Безопасность

* Оригинальные фотографии не удаляются, не изменяются, не перемещаются и не перезаписываются.
* Отзыв доступа выполняется только через системные настройки Android, потому что приложение не может программно снять выданное пользователем runtime-разрешение.
* Фотографии, координаты, EXIF и идентификаторы файлов не отправляются на сервер.

## APK

Ожидаемое имя release asset:

```text
photomap-v0.5.0.apk
```

Signed release APK нужно собрать после проверки release prep-ветки, когда `versionName=0.5.0` и `versionCode=6` находятся в коде.

## Проверки

* Release APK найден: `app/release/app-release.apk`.
* APK metadata checked via `aapt2 dump badging`:
  * package: `com.example.photomap`;
  * versionCode: `6`;
  * versionName: `0.5.0`;
  * minSdkVersion: `29`;
  * targetSdkVersion: `36`;
  * application label: `Photo Map`.
* APK permissions checked via `aapt2 dump permissions`.
* `.\gradlew.bat :app:compileDebugKotlin` - blocked before Kotlin compilation.

Причина блокировки: Gradle в sandbox не смог разрешить plugin artifact `org.gradle.toolchains.foojay-resolver-convention:1.0.0` из workspace cache, а сетевой запуск wrapper был отклонен.

Тесты, `npx`, `tsc` и `eslint` не запускались по правилу проекта.

## Известные ограничения

* Heatmap реализована bitmap-маркерами поверх текущей MapLibre annotations API; архитектура оставляет возможность позже перейти на style/source-based MapLibre layers.
* Экран настроек пока содержит только параметры, нужные для этой версии.
* WorkManager-фоновое сканирование, Coil-галерея и полноэкранный просмотр фотографий еще не реализованы.
