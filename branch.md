# Ветка `feature-04-large-gallery-performance`

## Цель

Оптимизировать поведение приложения для большой локальной галереи, ориентир - 40 000 фотографий: убрать зависание после обновления, сделать сканирование понятным по прогрессу и ограничить количество маркеров, которые карта строит одновременно.

## База ветки

Ветка создаётся от актуальной `main` после merge ветки `feature-03-map-screen`.

## Границы ветки

В ветку входит:

* быстрый автоматический MediaStore-скан без чтения EXIF всех файлов;
* отдельная ручная команда для глубокого поиска GPS в EXIF;
* прогресс сканирования с пакетным обновлением;
* отказ от Logcat-записи на каждую фотографию;
* хранение количества фотографий с координатами в `UiState`, без пересчёта по 40k элементов в Compose;
* групповые маркеры карты с числом фотографий;
* построение маркеров только для видимой области карты после первичного центрирования.

В этой ветке не реализуются:

* Room-индекс;
* полноценная галерея с Coil;
* фоновый WorkManager-сканер;
* постраничная загрузка из базы;
* миниатюры внутри маркеров.

## Задачи

1. Сканирование:
   * не читать EXIF автоматически при старте приложения;
   * оставить быстрый проход MediaStore для первичной загрузки;
   * добавить ручной deep scan для EXIF;
   * показывать прогресс обработки каждые 500 фотографий.

2. UI:
   * заменить общий текст загрузки на понятный статус;
   * показывать `Обработано X из Y`;
   * не пересчитывать количество GPS-фото в Compose при каждой recomposition.

3. Карта:
   * сохранять группировку точек по zoom bucket;
   * показывать число фотографий на групповых маркерах;
   * после первичного fit bounds строить маркеры только в текущих видимых границах карты.

4. Безопасность:
   * сохранить read-only доступ к MediaStore и EXIF;
   * не добавлять write/manage permissions;
   * не менять и не удалять оригинальные фотографии.

## Критерии готовности

Ветка считается готовой, если:

* проект собирается `assembleDebug`;
* приложение не зависает на авто-скане большой галереи;
* ручное EXIF-сканирование показывает прогресс;
* на карте групповые маркеры отображают количество фотографий;
* карта не строит все 40k маркеров одновременно после zoom/pan;
* в Git нет APK, keystore, секретов и артефактов сборки.

## Проверки

После реализации выполнить только `assembleDebug`. Тесты, `npx`, `tsc` и `eslint` не запускать без отдельного запроса пользователя.
