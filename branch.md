# Ветка `feature-02-permissions-mediastore`

## Цель

Реализовать первый рабочий слой доступа к фотографиям устройства: runtime-разрешения, чтение изображений через Android MediaStore и вывод найденных фотографий в UI/лог без сохранения в Room.

## База ветки

Ветка создаётся от актуальной `main` после merge bootstrap-проекта.

## Границы ветки

В ветку входит:

* обработка разрешений на чтение изображений для Android 10-14+;
* запрос `ACCESS_MEDIA_LOCATION`;
* базовая проверка состояния доступа;
* чтение списка изображений через MediaStore;
* доменная модель фотографии для этапа MediaStore;
* отдельный компонент чтения MediaStore вне Activity и ViewModel;
* ViewModel со StateFlow для экрана разрешений/медиатеки;
* минимальный Compose-интерфейс на русском;
* логирование найденных фотографий.

В этой ветке не реализуются:

* Room;
* чтение EXIF;
* сохранение локального индекса;
* карта и кластеризация;
* галерея с Coil;
* полноэкранный просмотр;
* WorkManager.

## Задачи

1. Разрешения:
   * добавить permissions в `AndroidManifest.xml`;
   * учитывать `READ_EXTERNAL_STORAGE` до Android 13;
   * учитывать `READ_MEDIA_IMAGES` на Android 13+;
   * учитывать частичный доступ `READ_MEDIA_VISUAL_USER_SELECTED` на Android 14+;
   * запросить `ACCESS_MEDIA_LOCATION` для исходных координат;
   * показать отказ и возможность повторного запроса.

2. MediaStore:
   * создать модель фотографии для считанных данных;
   * читать изображения через `ContentResolver` и `MediaStore.Images.Media`;
   * формировать URI через `ContentUris.withAppendedId`;
   * получать MediaStore ID, URI, имя, MIME-тип, даты, размеры, размер файла и ориентацию;
   * не использовать абсолютные пути как идентификатор;
   * логировать найденные фотографии.

3. Архитектура:
   * вынести чтение MediaStore в отдельный data-компонент;
   * не обращаться к `ContentResolver` напрямую из ViewModel;
   * держать UI-состояние во ViewModel через `StateFlow`;
   * оставить Activity только точкой входа Compose.

4. UI:
   * сделать экран с объяснением, зачем нужен доступ к фото;
   * добавить кнопку «Предоставить доступ»;
   * показать состояние отказа;
   * показать количество найденных фотографий;
   * показать небольшой список последних считанных фотографий;
   * добавить кнопку повторного сканирования.

## Критерии готовности

Ветка считается готовой, если:

* приложение не падает без разрешений;
* пользователь может запросить доступ к фотографиям;
* после выдачи доступа приложение читает изображения через MediaStore;
* найденные фотографии выводятся в лог;
* UI показывает количество найденных фотографий;
* ViewModel не обращается напрямую к MediaStore, DAO или `ContentResolver`;
* APK/артефакты сборки не попадают в Git.

## Проверки

Автоматические тесты и сборочные проверки в этой ветке не запускать после промпта, если пользователь не попросит отдельно.
