# Ветка `feature-03-map-screen`

## Цель

Добавить первый экран карты: Google Maps Compose, безопасную конфигурацию API key, чтение координат фотографий в режиме read-only и отображение найденных снимков с геопозицией на карте.

## База ветки

Ветка создаётся от актуальной `main` после merge ветки `feature-02-permissions-mediastore`.

## Границы ветки

В ветку входит:

* подключение Google Maps Compose;
* подключение Maps Compose Utils для кластеризации;
* чтение `MAPS_API_KEY` из `local.properties`;
* передача API key через manifest placeholder;
* экран карты на Jetpack Compose;
* пустое состояние для отсутствия координат или API key;
* чтение координат фотографий только в read-only режиме;
* отображение фотографий с координатами на карте;
* базовая кластеризация точек;
* сохранение строгого запрета на удаление/изменение оригинальных фотографий.

В этой ветке не реализуются:

* Room-индекс;
* полноценный фоновый scanner;
* галерея с Coil;
* полноэкранный просмотр;
* кастомные marker thumbnails;
* WorkManager;
* нижняя навигация.

## Задачи

1. Конфигурация карты:
   * добавить зависимости Google Maps Compose и Maps Compose Utils;
   * добавить чтение `MAPS_API_KEY` из `local.properties`;
   * добавить пример `local.properties.example`;
   * не хранить реальный API key в Git.

2. Координаты:
   * расширить модель фотографии координатами;
   * сначала использовать доступные MediaStore-колонки, если они есть;
   * если координат нет, читать EXIF через `ExifInterface`;
   * открывать изображения только через `ContentResolver`;
   * не открывать URI на запись;
   * считать координаты валидными только для диапазонов широты/долготы и не принимать `0,0`.

3. UI:
   * добавить экран карты;
   * показывать количество фотографий с координатами;
   * показывать состояние отсутствия координат;
   * показывать состояние отсутствия API key;
   * дать переход между экраном доступа/сканирования и картой;
   * при нажатии на marker показывать краткое название фотографии.

4. Безопасность:
   * не добавлять write/manage permissions;
   * не использовать MediaStore delete/trash/write requests;
   * не менять EXIF оригинальных файлов;
   * не копировать оригиналы фотографий.

## Критерии готовности

Ветка считается готовой, если:

* проект собирается `assembleDebug`;
* приложение запускается без API key и показывает понятное состояние;
* при наличии API key открывается карта;
* фотографии с валидными координатами отображаются на карте;
* фотографии без координат не отображаются на карте;
* чтение EXIF выполняется только read-only;
* в Git нет API key, APK, keystore и артефактов сборки.

## Проверки

После реализации выполнить только `assembleDebug`. Тесты, `npx`, `tsc` и `eslint` не запускать без отдельного запроса пользователя.
