# Traverse v0.3.1

Дата: 2026-07-15

Performance-релиз для больших локальных галерей. Основной ориентир - 40 000 фотографий: приложение не должно выглядеть зависшим после обновления и не должно строить все маркеры карты одновременно.

## Добавлено

* Отдельная ручная команда `Искать GPS в EXIF` для глубокого поиска координат.
* Прогресс сканирования MediaStore/EXIF с обновлением каждые 500 фотографий.
* Групповые маркеры карты с отображением количества фотографий.

## Изменено

* Автоматический скан после запуска больше не читает EXIF всех фотографий.
* Быстрый стартовый проход читает список фотографий через MediaStore без открытия каждого файла.
* Карта после первичного центрирования строит маркеры только для видимой области.
* Количество фотографий с координатами хранится в `UiState`, без пересчёта 40k элементов в Compose при каждой перерисовке.
* Убрано Logcat-логирование каждой фотографии во время сканирования.

## Безопасность

* Приложение продолжает работать только с локальными фотографиями через read-only доступ.
* Исходные фотографии не удаляются, не изменяются, не перемещаются и не перезаписываются.
* Фотографии, координаты, EXIF и идентификаторы файлов не отправляются на сервер приложения.

## Известные ограничения

* Room-индекс ещё не реализован.
* Результаты EXIF-сканирования пока не сохраняются между запусками в локальную базу.
* Галерея с миниатюрами ещё не реализована.
* Полноэкранный просмотр ещё не реализован.
* WorkManager-фоновое сканирование ещё не реализовано.
