# Traverse v0.4.0

Дата: 2026-07-15

Релиз локального индекса обработанных фотографий. Основная цель - не терять прогресс глубокого EXIF-сканирования и не перечитывать заново 40 000 фотографий после перезапуска приложения.

## Добавлено

* Локальная SQLite-база `photo_index.db` для индекса обработанных фотографий.
* Сохранение metadata фотографии, координат и статуса `locationScanned`.
* Сохранение прогресса глубокого EXIF-сканирования между запусками.
* Статистика GPS-индекса на экране доступа.

## Изменено

* Повторный deep scan переиспользует уже обработанные неизменённые фотографии.
* Cache hit определяется по `MediaStore ID`, `dateModified` и `size`.
* Переименование фотографии обновляет metadata в индексе без повторного EXIF-чтения, если файл не изменился.
* Записи удалённых из MediaStore фотографий удаляются только из локального индекса.
* Прогресс сканирования обновляется по времени и показывает процент.

## Безопасность

* Оригинальные фотографии не удаляются, не изменяются, не перемещаются и не перезаписываются.
* Удаление в этой версии касается только строк приватной SQLite-базы приложения.
* Фотографии, координаты, EXIF и идентификаторы файлов не отправляются на сервер приложения.
* APK не должен запрашивать write/manage-разрешения.

## APK

Ожидаемое имя release asset:

```text
photomap-v0.4.0.apk
```

Signed release APK нужно собрать после проверки этой release prep-версии, когда `versionName=0.4.0` и `versionCode=5` уже находятся в коде.

## Проверки

* `.\gradlew.bat assembleDebug` - passed.
* Debug APK metadata checked: `versionCode=5`, `versionName=0.4.0`.
* Debug APK permissions checked via `aapt2 dump permissions`.
* Source search for photo delete/write paths - checked.

## Известные ограничения

* Это пока lightweight SQLite-индекс, не полноценная Room-архитектура.
* Нет пользовательской кнопки очистки индекса в настройках.
* WorkManager-фоновое сканирование ещё не реализовано.
* Галерея с миниатюрами и полноэкранный просмотр ещё не реализованы.
