# Photo Map

Нативное Android-приложение для локальной галереи фотографий с будущим отображением снимков на карте.

Проект разрабатывается по этапам. Текущая ветка `feature-14-trip-segmentation` добавляет локальную кластеризацию поездок и отдельную темную карту поездок.

## Текущее состояние

Реализовано:

* Android-проект с Kotlin и Jetpack Compose;
* Material 3 тема;
* runtime-запрос разрешений на чтение фотографий;
* поддержка `READ_EXTERNAL_STORAGE` для Android 10-12;
* поддержка `READ_MEDIA_IMAGES` для Android 13+;
* поддержка частичного доступа `READ_MEDIA_VISUAL_USER_SELECTED` для Android 14+;
* запрос `ACCESS_MEDIA_LOCATION`;
* чтение изображений через `MediaStore.Images.Media`;
* формирование URI через `ContentUris.withAppendedId`;
* вывод найденных фотографий в Logcat с тегом `PhotoMapMediaStore`;
* минимальный экран на русском с количеством найденных фотографий;
* read-only чтение GPS-координат из MediaStore/EXIF;
* первый экран карты на MapLibre без Google Maps API key;
* отображение фотографий с координатами на карте;
* базовая группировка близких точек;
* быстрый авто-скан без чтения EXIF всех файлов;
* ручной поиск GPS в EXIF с прогрессом;
* групповые маркеры с числом фотографий;
* локальный SQLite-индекс обработанных фотографий;
* повторное использование координат для неизменённых фотографий;
* overlay-маркеры карты с миниатюрами и кластерами;
* debug-панель карты с переключателем в настройках;
* Navigation Compose для экранов приложения;
* локальная сегментация поездок по проиндексированным фотографиям с датой и геопозицией;
* отдельная темная карта поездок с маркерами-миниатюрами и бейджем `+N`;
* страница поездки с местами маршрута, датами, сглаженной линией маршрута и grid-галереей фотографий поездки.

Пока не реализовано:

* полноценный Room-индекс;
* галерея с миниатюрами;
* полноэкранный просмотр;
* WorkManager-фоновое сканирование.

## Стек

* Kotlin;
* Jetpack Compose;
* Material 3;
* Android MediaStore;
* ExifInterface;
* MapLibre Android SDK;
* Navigation Compose;
* Kotlin Coroutines;
* Flow и StateFlow;
* ViewModel.

Следующие этапы добавят Room, Coil, WorkManager и Hilt. MapLibre с OpenFreeMap не требует Google Maps API key, Google Play Services, регистрации или billing, но онлайн-карта требует интернет и доступный style/tile URL.

## Требования

* Android Studio с поддержкой Gradle Kotlin DSL;
* Android SDK для compileSdk проекта;
* устройство или эмулятор с Android 10+.

## Запуск

Открой проект в Android Studio и запусти конфигурацию `app`.

Команда сборки:

```bash
./gradlew assembleDebug
```

На Windows:

```powershell
.\gradlew.bat assembleDebug
```

## Разрешения

Приложение запрашивает:

* `READ_EXTERNAL_STORAGE` на Android 10-12;
* `READ_MEDIA_IMAGES` на Android 13+;
* `READ_MEDIA_VISUAL_USER_SELECTED` на Android 14+;
* `ACCESS_MEDIA_LOCATION` для будущего чтения исходных геоданных из медиа.

Если доступ к фотографиям не выдан, приложение не должно падать. После отказа доступ можно запросить повторно или открыть системные настройки приложения.

## MediaStore

Текущая реализация читает:

* MediaStore ID;
* URI;
* имя файла;
* MIME-тип;
* дату добавления;
* дату изменения;
* дату съёмки;
* ширину;
* высоту;
* размер файла;
* ориентацию.

Абсолютные пути файлов не используются как идентификаторы. Основной идентификатор - MediaStore ID.

## Локальный индекс

Приложение хранит локальную SQLite-базу `photo_index.db` с metadata уже обработанных фотографий. Индекс нужен только для ускорения повторного сканирования и восстановления прогресса.

Для проверки актуальности записи используются `MediaStore ID`, `dateModified` и `size`. Если фотография переименована, metadata обновляется. Если фотография удалена из MediaStore, удаляется только запись из локального индекса.

## Карта

Карта работает через MapLibre Android SDK.

После первичного центрирования по всем фотографиям с координатами карта строит маркеры только для текущей видимой области. Это нужно, чтобы не рисовать десятки тысяч маркеров одновременно на больших галереях. Видимые маркеры рисуются Compose-overlay поверх MapLibre, чтобы кластеры и миниатюры оставались видимыми независимо от особенностей style layer rendering.

По умолчанию используется OpenFreeMap Liberty style:

```properties
MAP_STYLE_URL=https://tiles.openfreemap.org/styles/liberty
```

Карта поездок использует отдельный темный OpenFreeMap style:

```properties
TRIP_MAP_STYLE_URL=https://tiles.openfreemap.org/styles/dark
```

При необходимости URL стилей можно переопределить в `local.properties`. Если позже используется платный tile provider с ключом, этот ключ нельзя добавлять в Git.

## Поездки

Основная карта содержит FAB `Поездки`. Он открывает отдельный экран карты поездок, визуально отделенный темной темой и темными тайлами MapLibre.

Поездки считаются локально по уже проиндексированным фотографиям с валидными координатами и датой. Приложение строит фотосессии, определяет базовые регионы и выделяет away-сегменты. На карте поездка отображается одним маркером-миниатюрой с бейджем `+N`, где `N` - количество фотографий в поездке.

Нажатие на маркер открывает страницу поездки: сверху отображается темная карта со сглаженной хронологической линией маршрута от первой фотографии поездки к последней, снизу - grid-галерея фотографий этой поездки. Миниатюры на точках маршрута кликабельны и прокручивают нижнюю галерею к выбранному снимку. Клик по фото в нижнем grid открывает снимок в системной галерее Android.

Заголовок страницы поездки показывает ближайший населенный пункт или несколько пунктов маршрута, если системный geocoder доступен. Даты поездки отображаются второй строкой. Действие `Центр` на странице поездки сбрасывает zoom и центр карты на весь маршрут поездки. При возврате со страницы поездки карта поездок сохраняет zoom и центр выбранной поездки.

На карте поездок справа расположен минимальный вертикальный scrubber поездок: без рамки и фоновой карточки, только полупрозрачные отметки. Он выглядит как длинный timeline, может выходить за границы экрана, держит выбранную точку около видимой зоны и снапится к ближайшей точке поездки при tap/drag. Активная поездка подсвечивается акцентным цветом и рамкой, соседние точки остаются приглушенными; если поездка одна, отображается только точка без линии. Подписи поездок расположены сбоку от точек с прозрачностью около 33%.

На странице поездки справа расположен такой же минимальный scrubber фотографий: tap/drag выбирает снимок по хронологии, центрирует карту маршрута на точке фотографии и прокручивает нижний grid к выбранному снимку.

## Конфиденциальность

Все данные остаются на устройстве. Приложение не отправляет фотографии, координаты, EXIF или идентификаторы файлов на сервер.
