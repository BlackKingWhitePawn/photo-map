# Ветка `feature-05-persistent-photo-index`

## Цель

Сделать локальный индекс уже обработанных фотографий, чтобы прогресс глубокого EXIF-сканирования не терялся между запусками приложения и чтобы повторный запуск не перечитывал заново 40 000 фотографий.

## База ветки

Ветка создаётся от актуальной `main` после merge ветки `feature-04-large-gallery-performance`.

## Границы ветки

В ветку входит:

* локальная SQLite-база `photo_index.db`;
* сохранение обработанных фотографий по `MediaStore ID`;
* сохранение ревизии файла через `dateModified` и `size`;
* повторное использование координат и статуса EXIF-обработки для неизменённых фотографий;
* возобновляемый прогресс глубокого EXIF-сканирования;
* обновление записи при переименовании фотографии;
* удаление из локального индекса записей, которых больше нет в MediaStore;
* пакетное сохранение маленькими партиями, чтобы при остановке повторялась только небольшая часть работы.

В этой ветке не реализуются:

* полноценный Room-слой;
* фоновый WorkManager-сканер;
* галерея с Coil;
* ручное управление базой из настроек;
* миграции пользовательских данных сложнее версии `1`.

## Задачи

1. Локальная база:
   * создать таблицу `photos`;
   * хранить metadata фотографии, координаты и флаг `locationScanned`;
   * хранить `indexedAt`;
   * добавить статистику индекса.

2. Сканирование:
   * перед чтением EXIF проверять запись в локальном индексе;
   * если `mediaId`, `dateModified` и `size` совпадают, использовать сохранённые координаты;
   * если файл новый или изменён, читать MediaStore/EXIF заново;
   * сохранять прогресс партиями до 50 фотографий;
   * обновлять UI progress по времени, а не шагом 500.

3. Переименование и удаление:
   * при совпавшем `mediaId` обновлять `displayName`, URI и прочие metadata;
   * если фото исчезло из MediaStore, удалить только запись из локальной базы;
   * не удалять, не менять и не перемещать оригинальный файл.

4. UI:
   * показывать сохранённый статус GPS-индекса;
   * показывать нормальный progress bar и процент;
   * после перезапуска продолжать с уже сохранённого индекса.

## Критерии готовности

Ветка считается готовой, если:

* проект собирается `assembleDebug`;
* повторный deep scan пропускает уже обработанные неизменённые фотографии;
* переименованная фотография обновляет metadata в индексе;
* удалённая из MediaStore фотография исчезает из локального индекса;
* прогресс не сбрасывается полностью после перезапуска;
* в Git нет APK, keystore, секретов и артефактов сборки.

## Проверки

После реализации выполнить только `assembleDebug`. Тесты, `npx`, `tsc` и `eslint` не запускать без отдельного запроса пользователя.
