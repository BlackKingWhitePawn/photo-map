# Traverse v0.10.0

Date: 2026-07-18

## Added

* Two-level trip heatmap built from detected trips and destinations instead of raw photo counts.
* Local trip heatmap tables in `photo_index.db` for aggregated heat cells, per-trip contributions, and heatmap metadata.
* H3-backed heat aggregation at resolutions 4, 5, 6, 7, 8, and 9 with a deterministic local fallback grid.
* Trip heatmap intensity based on trip count, days spent, active days, and photo session count.
* Background `TripHeatmapWorker` scheduling through WorkManager after media permissions are available.
* GeoJSON mapping for ready heat cells and a MapLibre `HeatmapLayer` rendered below photo clusters and trip markers.
* Visible heatmap diagnostics in the map debug panel and settings diagnostics.

## Changed

* Main map and trip map request ready heatmap cells for the current viewport and zoom instead of calculating heat data during rendering.
* Opening the trip map from the normal map preserves the current map center and zoom instead of resetting the camera.
* Android version metadata is now `versionName=0.10.0`, `versionCode=16`.

## Fixed

* Trip heatmap layer is attached to both the main map and the trip map, so the heatmap remains visible behind trip markers.
