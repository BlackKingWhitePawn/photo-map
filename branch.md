# Ветка `feature-06-map-heatmap-controls`

## Цель

Добавить новый режим отображения карты с heatmap-метками, настраиваемым порогом появления фото-миниатюр и управлением длительными действиями сканирования.

## База ветки

Ветка создана от актуальной `main` после релиза `v0.4.0`.

## Границы ветки

В ветку входит:

* heatmap-отображение групп фотографий на MapLibre-карте;
* показ миниатюры представительной фотографии, когда группа достигает настраиваемого порога;
* подпись миниатюрного маркера в формате `+N фото`;
* экран настроек с изменением порога heatmap-миниатюр;
* сохранение настройки порога в локальных `SharedPreferences`;
* пауза, продолжение и отмена текущего индексирования;
* публикация уже проиндексированных batch-результатов в UI во время сканирования;
* отображение карты во время активного индексирования вместо пустого состояния `нет фотографий`;
* кнопка отзыва доступа через системные настройки приложения;
* нормальная иконка настроек для кнопок перехода в настройки;
* обработка системного события `назад` через локальный стек экранов.

В ветку не входит:

* полноценная Room-миграция;
* WorkManager-фоновое сканирование;
* полноэкранный просмотр фото;
* Coil-галерея;
* публикация APK/AAB в GitHub Releases;
* программный отзыв Android-разрешений без системных настроек, так как платформа не позволяет приложению снимать выданные пользователем runtime-разрешения напрямую.

## Реализация

1. Карта:
   * заменить чистые точки на heatmap-подобные bitmap-маркеры;
   * группировать точки по сетке в зависимости от zoom bucket;
   * для групп выше порога использовать маркер с миниатюрой и подписью;
   * сохранять возможность позже заменить bitmap-маркеры на style/source-based MapLibre слои.

2. Настройки:
   * добавить отдельный экран настроек;
   * показать статистику индекса;
   * добавить управление порогом миниатюр;
   * добавить запуск сканирования и deep scan из настроек;
   * добавить паузу, продолжение и отмену активного сканирования.

3. Сканирование:
   * добавить кооперативный control для паузы;
   * отменять текущую coroutine без удаления уже сохраненного индекса;
   * обновлять `UiState` batch-результатами, чтобы карта наполнялась по мере обработки.

4. Доступ:
   * менять текст основной кнопки доступа между `Предоставить доступ` и `Отозвать доступ`;
   * вести на системный экран приложения для отзыва доступа;
   * повторно проверять разрешения при возврате в приложение.

## Проверки

Сборку и compile checks выполняет пользователь вручную. Codex не запускает build/compile после патчей и в итоговом отчёте явно указывает, что сборка оставлена пользователю.

Ручная команда для проверки пользователем:

```bash
.\gradlew.bat assembleDebug
```

Тесты, `npx`, `tsc` и `eslint` не запускать без отдельного запроса пользователя.
